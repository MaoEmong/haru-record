# 오늘 탭 시간 탐색 Deep Interview 결과

날짜: 2026-04-29
주제: 오늘 탭 재생바를 이용한 하루 위치 기록 탐색

## 문제 정의

오늘 탭에는 하루 시간 흐름을 보여주는 재생바가 있지만 현재는 시각적 진행 표시 역할에 가깝다. 사용자는 이 재생바를 드래그해서 오늘 찍힌 위치 기록을 시간순으로 탐색하고, 선택한 시점 기준으로 지도와 위치 카드가 바뀌는 경험을 원한다.

## 대상 사용자

- 오늘 하루 동안 어디에 있었는지 빠르게 되짚어보고 싶은 사용자
- 지도와 현재 위치/방문 흐름을 시간순으로 확인하려는 사용자

## 핵심 요구사항

1. [P0] 오늘 탭의 재생바를 드래그 가능하게 만든다.
2. [P0] 재생바는 오늘 찍힌 실제 위치 기록 시점 단위로 스냅된다.
3. [P0] 선택한 시점 기준으로 상단 지도와 현재 위치 카드가 바뀐다.
4. [P0] 과거 시점을 보고 있을 때 `현재로` 버튼/텍스트를 표시한다.
5. [P0] `현재로`를 누르면 현재 기록 상태로 정확히 돌아온다.
6. [P1] 지도는 하루 전체 경로를 희미하게 깔고, 선택 시점 위치를 강조한다.
7. [P1] 선택한 시점의 현재 위치 카드는 장소명과 기록 시간을 함께 보여준다.
8. [P1] 단순 경유 좌표는 `이동 중`과 기록 시간으로 표시한다.
9. [P1] 오늘 방문한 곳 리스트는 선택 시점까지 등장한 항목만 표시한다.
10. [P2] 오늘 탭을 나갔다가 돌아오면 선택 상태는 현재 시점으로 초기화한다.

## 제약 조건 & 전제

- 1차 범위는 오늘 탭만이다.
- 오늘 기록/하루 상세 화면의 시간 탐색은 이번 범위 밖이다.
- 하단 플레이어 컨트롤 버튼은 이번 작업에서 기능을 넣지 않는다. 추후 별도 기능을 넣을 수 있도록 그대로 둔다.
- 상단 거리/방문 수 요약은 선택 시점 기준으로 바꾸지 않고 하루 전체 요약을 유지한다.
- 좌표가 없으면 기존 빈 상태를 유지한다.
- 좌표가 1개만 있으면 드래그 의미는 없지만 해당 기록 위치/시점은 표시한다.

## 엣지 케이스 & 오류 시나리오

- 오늘 위치 기록이 0개인 경우
- 오늘 위치 기록이 1개뿐인 경우
- 선택한 좌표가 저장된 장소/새 장소 후보가 아닌 단순 경유 좌표인 경우
- 사용자가 과거 시점을 선택한 뒤 다른 탭으로 이동했다 돌아오는 경우
- 경로 데이터는 있으나 방문 기록은 아직 없는 경우
- 표시용 경로 정리로 원본 좌표보다 표시 좌표가 줄어든 경우

## 범위 밖

- 오늘 기록 화면 시간 탐색
- 하루 상세 화면 시간 탐색
- 자동 재생 기능
- 하단 이전/일시정지/다음 버튼 기능 변경
- 선택 시점 기준 거리/방문 수 재계산
- 과거 날짜 탐색 UX

## 수용 기준

- [ ] 재생바를 드래그하면 오늘 위치 기록 지점 단위로 선택 시간이 바뀐다.
- [ ] 선택 시간 라벨은 기존 왼쪽 시간 라벨에 표시된다.
- [ ] 선택한 시점에 맞춰 지도 위치 강조가 바뀐다.
- [ ] 선택한 시점에 맞춰 현재 위치 카드가 장소명/`이동 중` + 기록 시간으로 바뀐다.
- [ ] 오늘 방문한 곳 리스트는 선택 시점까지 등장한 항목만 보여준다.
- [ ] 과거 시점을 보고 있을 때 `현재로` 컨트롤이 보인다.
- [ ] `현재로`를 누르면 현재 기록 상태로 정확히 돌아온다.
- [ ] 오늘 탭을 나갔다 돌아오면 현재 시점 상태로 초기화된다.
- [ ] 좌표가 없는 경우 기존 빈 상태가 깨지지 않는다.
- [ ] 좌표가 1개뿐인 경우 해당 위치 지도와 카드가 표시된다.

## 구현 계획으로 넘길 메모

- 현재 오늘 탭 진행바는 `HomeScreen._todayProgress()`와 `home_player_widgets.dart`의 `_ProgressBar`가 담당한다.
- 오늘 탭 지도는 `home_album_art.dart`의 `_AlbumArtSection`과 `DayRouteSnapshot`을 기반으로 한다.
- 현재 위치/오늘 방문한 곳은 `home_timeline_sections.dart`의 `_CurrentLocationCard`, `_TodayVisitList`가 담당한다.
- 구현 계획 단계에서는 `DayRouteSnapshot.points`와 `DayActivityPreview.timeline`을 시간 선택 상태에 맞게 필터링/선택하는 얇은 모델을 먼저 설계해야 한다.
